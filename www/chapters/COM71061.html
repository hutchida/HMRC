COM71061

Enquiry Into Return For The Wrong Period (Action Guide)

0

To amend the AP record of a company that has made a return for the wrong period before you issue a closure notice, consider steps 1 - 13 below. The guide is presented as follows.

1

Revising the AP record

2

1. If you are ready to issue the closure notice to end an enquiry into a return made for the wrong period, whether with COTAX or manually, you need to amend the company’s AP record to the correct period or periods before you issue it. Follow the steps below.

3

2. Use function MRSC (Maintain Repay / Realloc Signals (Company)) to inhibit automatic repayments and reallocations for all APs.

4

3. Use function MAPS (Maintain AP Signals) to change the ‘in progress’ signal to ‘N’ on the AP that you want to supersede.

5

 You must:

6

print the return and attachments from View CT Returns or for a paper return, use the [Print] button in function DRTN (Display Return)

7

use the [Print] button in function DASS (Display Assessment) to print copies of all assessments on the AP that you want to supersede.

8

Keep them in case you need them later for evidential purposes as the information disappears from the IT systems when you use function URTN (Unlog Return).

9

If you have made a jeopardy amendment for the period under enquiry, contact the CT Business Help Desk(This content has been withheld because of exemptions in the Freedom of Information Act 2000) before you unlog the return.

10

5. Use function URTN (Unlog Return) to unlog the return on the AP that you want to supersede.

11

6. Use function MAPD (Maintain AP Dates) to supersede the period that is wrong and replace it with the correct period or periods.

12

7. Use functions LRTN (Log Return) and CRTN (Capture Return) to:

13

log and capture the return and record the self assessment on the correct AP, using the original date of receipt of the return

14

change the ‘Do you wish to issue an acknowledgement?’ field entry from ‘Y’ to ‘N’ in the final screen of CRTN (COT112G).

15

8. Use function MAPS for the correct AP to set the ‘enquiry in progress’ signal to ‘Y’. In the ‘started’ field, delete today’s date and enter the original enquiry start date. Enter the enquiry type in the ‘type’ field.

16

9. Use function MRSC to unset the inhibition on automatic repayments and reallocations for all APs.

17

If the superseded and superseding APs are:

18

linked, use function VPPD (View Payment and Posting Details) to confirm that any payments have moved correctly from the superseded to the superseding AP

19

not linked, use function REWD (Direct Reallocation Within COTAX) to move payments shown as stranded in VPPD on the superseded AP to the superseding AP.

20

Issue the closure notice:

21

using function RAMA (Record / Amend Assessment) wherever possible or

22

manually if the wording of the standard COTAX closure notice does not suit your case. See COM71093 for more information.

Interaction with Caseflow or CQI

25

Caseflow and CQI are not automatically updated from COTAX.

26

If your office uses these systems, you must update them to reflect all of the changes you have made to the COTAX record.
